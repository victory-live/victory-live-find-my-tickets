# Victory Live · Ticket Finder
## File Structure

```
victorylive-ticket-finder/
├── index.html     — Page structure and markup (3 KB)
├── styles.css     — All styling, colors, layout, responsive rules (8 KB)
├── app.js         — All JavaScript logic: category rendering, search, card rendering (9 KB)
├── data.js        — All ticket data as a JS module (window.__VL_DATA__) loaded before app.js (2.3 MB)
└── data.json      — Same data as data.js but as pure JSON — useful for a backend/CMS migration
```

---

## How It Works

1. `index.html` loads `styles.css`, then `data.js`, then `app.js` in that order.
2. `data.js` populates `window.__VL_DATA__` with all 8 categories of ticket data.
3. `app.js` reads that data object and drives the entire UI (category grid, search, result cards).

---

## Hosting

Drop all 4 files into the same directory on any static web host (Netlify, Vercel, S3, Apache, Nginx, etc.). No server-side code or database required — everything runs client-side.

**Recommended:** For production, consider moving the data out of `data.js` and into a proper API endpoint or CMS so updates don't require a full file replacement.

## Updating the Data

1. Edit `data.json` with new/changed entries.
2. Run this one-liner to regenerate `data.js`:
   ```bash
   echo "window.__VL_DATA__ = $(cat data.json);" > data.js
   ```
3. Re-deploy both `data.js` and `data.json`.
